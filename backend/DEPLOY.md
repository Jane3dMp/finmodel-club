# Развёртывание бэкенда — пошаговая инструкция

> Для Claude Code. Все команды и проверки — по порядку, ничего не пропускать.

---

## ЧТО ПОЛУЧИТСЯ

Было: клиент на GitHub Pages + Google Apps Script + Google-таблица (медленно, ломается).
Станет: клиент + PHP API + MySQL на белорусском хостинге (быстро, надёжно, 24/7).

---

## ЧТО НУЖНО ОТ ЖАННЫ (получить до старта)

| Что | Где взять | Пример |
|---|---|---|
| Домен/поддомен | панель хостинга | `model.proznanie.by` |
| Доступ FTP или SSH | письмо от хостера | хост, логин, пароль |
| Параметры MySQL | панель хостинга | хост, имя БД, логин, пароль |
| HTTPS включён | панель хостинга | Let's Encrypt, обычно бесплатно |

⚠️ Пароли передавать безопасным способом (не открытым текстом в чате).

## ШАГ 1. Проверить хостинг

Требования:
- PHP **8.0 или выше** (проверить: создать `phpinfo.php` с `<?php phpinfo();`, открыть, посмотреть версию, **потом удалить файл**).
- MySQL 5.7+ / MariaDB 10.3+.
- Расширения PHP: `pdo_mysql`, `json` (обычно есть по умолчанию).
- HTTPS работает.

Если PHP старше 8.0 — попросить хостера переключить (обычно кнопка в панели).

---

## ШАГ 2. Создать базу

1. В панели хостинга создать базу MySQL, записать: имя БД, логин, пароль, хост.
2. Через phpMyAdmin (есть у любого хостера) выполнить `schema/schema.sql`.
3. Проверить: должно появиться 9 таблиц —
   `versions`, `courses`, `lessons`, `teachers`, `fix_rates`, `wage_tiers`, `expenses`, `expenses_pct`, `plan_items`, `settings`.

---

## ШАГ 3. Залить бэкенд

Структура на хостинге:

```
/public_html/            (или /www, /htdocs — зависит от хостера)
  index.html             ← клиент
  /api/
    config.php           ← создать из config.example.php
    db.php
    load.php
    save.php
    migrate.php          ← удалить после миграции
```

1. Скопировать `config.example.php` → `config.php`.
2. Заполнить `config.php`: параметры MySQL и `auth_token`.
   Токен сгенерировать: `openssl rand -hex 24` (или любая случайная строка 32+ символов).
3. Залить папку `/api/` на хостинг.
4. **Проверить синтаксис** перед заливкой: `php -l api/db.php`, `php -l api/load.php`, `php -l api/save.php`, `php -l api/migrate.php`.

---

## ШАГ 4. Защитить config.php

Создать `/api/.htaccess`:

```apache
# Запретить прямой доступ к конфигу
<Files "config.php">
    Require all denied
</Files>

# Запретить листинг папки
Options -Indexes
```

Проверить: открыть `https://ваш-домен/api/config.php` → должно быть 403, а не текст файла.

---

## ШАГ 5. Проверить API

Открыть в браузере:

```
https://ваш-домен/api/load.php?token=ВАШ_ТОКЕН
```

Ожидаемо на пустой базе:
```json
{"ok":true,"data":{"scenarios":{},"current":"","savedAt":""}}
```

Если ошибка — смотреть логи PHP в панели хостинга.

Проверить защиту: открыть без токена → должно быть `{"ok":false,"error":"Доступ запрещён"}`.

---

## ШАГ 6. Перенести данные

1. Убедиться, что Жанна **сохранила все нужные версии** в Google-таблицу (нажала «Сохранить» на каждой).
2. Открыть в браузере:
   ```
   https://ваш-домен/api/migrate.php?token=ВАШ_ТОКЕН
   ```
3. Скрипт выведет отчёт: какие версии перенесены, какие пропущены (повреждённые).
4. **Сверить числа**: количество курсов, занятий, педагогов до и после должно совпадать.

⚠️ Часть версий в Google-таблице **повреждена** (обрывки вида `{"courses":[{"nar 1Z"}]}`) — скрипт их пропустит, это правильно. Целые на момент написания: «Основной», «Не трогать», «26/27 от 6.07.2026 (копия)(копия)».

5. **Удалить `migrate.php` с хостинга** после успешной миграции.

## ШАГ 7. Переключить клиент

В `index.html` найти:

```javascript
gasUrl: "https://script.google.com/macros/s/AKfycb.../exec"
```

Заменить на адрес API и добавить токен. Нужно поправить три места, где идут запросы:

### 7.1 Загрузка (автозагрузка при старте)

Было:
```javascript
fetch(S.gasUrl+"?action=load").then(r=>r.text())...
```

Стало:
```javascript
fetch(API_URL+"/load.php", {headers:{"X-Auth-Token":API_TOKEN}}).then(r=>r.text())...
```

### 7.2 Сохранение (`pushGas`)

Было:
```javascript
fetch(S.gasUrl,{method:"POST",headers:{"Content-Type":"text/plain;charset=utf-8"},
  body:JSON.stringify({action:"save",data:payload})})
```

Стало:
```javascript
fetch(API_URL+"/save.php",{method:"POST",
  headers:{"Content-Type":"application/json","X-Auth-Token":API_TOKEN},
  body:JSON.stringify({action:"save",data:payload})})
```

### 7.3 Константы

В начало скрипта добавить:
```javascript
const API_URL   = "https://ваш-домен/api";
const API_TOKEN = "ВАШ_ТОКЕН";  // тот же, что в config.php
```

**Важно:** форматы запроса и ответа специально сделаны такими же, как у Apps Script — вся остальная логика клиента работает без изменений.

⚠️ Токен в клиенте виден любому, кто откроет исходник страницы. Для внутреннего инструмента одного человека это приемлемо, но если нужна серьёзная защита — добавить HTTP Basic Auth через `.htaccess` на весь сайт.

## ШАГ 8. Проверка (обязательно всё)

- [ ] Модель открывается **менее чем за 2 секунды** (было 30–40 сек).
- [ ] Индикатор синхронизации зелёный.
- [ ] Все версии на месте, переключаются.
- [ ] Данные целые: курсы, расписание, педагоги, расходы.
- [ ] Кнопка «Сохранить» работает, ошибок нет.
- [ ] **Открыть с другого компьютера** → те же данные.
- [ ] Изменить что-то на компьютере А → сохранить → обновить на компьютере Б → изменение видно.
- [ ] **Расчёты не изменились**: сверить оборот/мес и ФОТ до и после миграции (должны совпасть до рубля).

## ШАГ 9. Отключить старое

Только после успешной проверки:
- Отключить веб-приложение Apps Script (Deploy → Manage deployments → Archive).
- Google-таблицу **оставить как архив**, не удалять.

## ЕСЛИ ЧТО-ТО ПОШЛО НЕ ТАК

| Симптом | Причина | Что делать |
|---|---|---|
| 500 на любом запросе | нет `config.php` или неверные данные БД | проверить конфиг, посмотреть логи PHP |
| «Доступ запрещён» | токен не совпадает | сверить токен в `config.php` и в `index.html` |
| Пустой ответ | ошибка PHP | включить `display_errors` временно, посмотреть |
| CORS-ошибка в консоли | клиент на другом домене | добавить домен в `allowed_origins` в конфиге |
| Данные не перенеслись | версии в Google повреждены | проверить отчёт `migrate.php`, взять версии у Жанны заново |

## ЧТО НЕ ЛОМАТЬ

1. **Форматы API** — они специально совместимы с Apps Script.
2. **Формулу оборота**: цена × дети × группы × визиты × недели.
3. **Две модели ЗП** (KPI и Фикс).
4. **Базу взносов**: ФОТ педагогов + админ-команда − НПД.
5. **Метку времени версии** — ставить только при реальном сохранении.
