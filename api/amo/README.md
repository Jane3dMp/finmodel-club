# Прокси к amoCRM (`api/amo/`)

Тянет заявки (сделки) из amoCRM в финмодель: выбрали воронку и этапы — получили список детей,
которых ещё нет в системе, и добавили их в «Список ожидания» с телефоном.

**Только чтение.** Прокси не создаёт и не меняет в amoCRM ничего.

## Что нужно сделать один раз (Жанна)

1. **Создать интеграцию в amoCRM** (≈3 минуты):
   amoCRM → **Настройки** → **Интеграции** → «**+ Создать интеграцию**» → «**Внешняя интеграция**».
   - Название любое, например «Финмодель».
   - Права доступа: достаточно **чтения сделок и контактов**.
   - Сохранить → вкладка «**Ключи и доступы**» → скопировать «**Ключ долгосрочного доступа**».
     Он длинный (несколько сотен символов) — копировать целиком.

2. **Создать файл `config.php` на сервере** (файловым менеджером хостинга), в папке
   `app.proznanie.club/finmodel/api/amo/`, по образцу `config.example.php`:

   ```php
   <?php
   return [
       'subdomain'    => 'ваш_поддомен',        // «proznanie» или «proznanie.amocrm.ru»
       'access_token' => 'длинный_токен_сюда',
   ];
   ```

   ⚠️ Токен — секрет. В git он не попадает (`config.php` в `.gitignore`), автозаливка его
   не удаляет (`**/config.php` в `exclude` у deploy.yml), через веб он не читается (`.htaccess`).
   Присылать токен в чат не нужно — вставьте его сразу в файл на сервере.

3. Готово. В админке: вкладка «**Клиенты**» → кнопка «**📥 Новые из amoCRM**».

Права доступа (кому можно дёргать прокси) берутся из `api/alfa/config.php` —
`firebase_project` / `allowed_emails` / `allowed_origins` общие с альфовским прокси,
чтобы списки не разъезжались.

## Действия

| Действие | Что делает |
|---|---|
| `?action=ping` | Проверка: токен принят, аккаунт виден |
| `?action=pipelines` | Воронки и их этапы (id, название, цвет, порядок) |
| `?action=leads` | Сделки выбранных этапов + контакты (имя, телефон, все поля) |

`leads` принимает `{pipelineId, statusIds:[…], createdFrom?}`. Пагинация по 250, максимум
10 страниц (2500 сделок) за запрос — предохранитель от зависания.

## Известные особенности amoCRM

- **HTTP 204 = «ничего не найдено»**, а не ошибка. Пустой этап отвечает именно так.
- **Имя ребёнка лежит у всех по-разному**: у кого-то в названии сделки, у кого-то в контакте,
  у кого-то в своём поле. Поэтому прокси отдаёт `fields` (все кастомные поля сделки и контакта),
  а источник имени выбирается в модалке на живых данных. Первая сделка и первый контакт
  печатаются в консоль браузера (`📥 amo · образец сделки`) — по ним и настраиваем.
- **Телефон** берётся из контакта (`field_code = PHONE`), первый непустой.
- Токен долгосрочный, но не вечный. Если появится ошибка «amoCRM не принял токен (401)» —
  выпустить новый ключ в той же интеграции и заменить в `config.php`.
