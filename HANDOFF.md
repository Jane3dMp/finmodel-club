## Что это за проект

Одностраничное веб-приложение — интерактивная финмодель + расписание детского образовательного клуба. **Вся система = один файл `index.html`** (интерфейс + логика инлайн, Firebase с CDN).

- **Git:** настроен как `Proznanie` / `proznanie.mogilev@gmail.com`.

  - `https://app.proznanie.club/finmodel` — основной (хостинг HostFly);
  - `https://jane3dmp.github.io/finmodel-club/` — запасной (GitHub Pages).
- **Деплой:** `git push` в `main` → GitHub Pages обновляется сам **+** GitHub Actions (`.github/workflows/deploy.yml`, SamKirkland/FTP-Deploy-Action) заливает по FTPS на хостинг. Секреты `FTP_HOST/FTP_USER/FTP_PASSWORD` — в GitHub Secrets.
- ⚠️ **Перед каждым коммитом:** извлечь `<script>` из `index.html` и прогнать `node --check`. Сломанный синтаксис = белый экран у клиента.

## Стек данных / синхронизация (СДЕЛАНО, работает)

- **Firebase Realtime Database** (проект `proznanie-finmodel`, регион `europe-west1`, план Spark/бесплатный) = **живой источник правды**, синхронизация в реальном времени между несколькими компами (цель — 5 одновременно). `FB_CONFIG` вшит в `index.html` (публичный, не секрет).
- **Модель данных в Firebase:** `scenarios/{имя}/grid/{lessonId}` — занятия по одному (гранулярно, у каждого занятия стабильный `id` → правки в разные занятия не затирают друг друга); `scenarios/{имя}/rest` — всё остальное одним JSON-блобом (иначе имена-ключи с точками, напр. курс «3D Unity 2.0», ломают запись в Firebase).
- **Google-таблица** (Apps Script, `S.gasUrl`) — теперь только ручной бэкап по кнопке «Сохранить». **localStorage** — локальный кэш. Есть автосохранение + предупреждение `beforeunload` при несохранённом.
- Ключевые функции: `fbStart`(auth-гейт) → `fbConnect` → `fbPush`/`fbSchedulePush`, `onRemoteModel`/`_applyRemote`, `_gridMap`/`_restStr`, `_ensureLessonIds`.

## СИНХРОНИЗАЦИЯ С ALFACRM — ЭТАП 1 СДЕЛАН (контакты детей), ждёт ключ на сервере

**Уточнённая цель (от Жанны, июль 2026):** дети уже занесены в модель вручную. Их
**контакты (телефоны родителей)** лежат в Альфе. Нужно сопоставить детей модели с
учениками Альфы и подтянуть телефоны — чтобы дальше **рассылать родителям сообщения
из админ-панели о запуске групп** по времени из расписания.

**Что построено и запушено (коммит «Синхронизация контактов детей из AlfaCRM»):**
- **PHP-прокси `api/alfa/`** (в КОРНЕ репо, не в `backend/` — иначе не зальётся; `backend/`
  исключён в deploy.yml). Заливается автозаливкой на `https://app.proznanie.club/finmodel/api/alfa/`.
  - `index.php` — роутер: `?action=ping|branches|customers`.
  - `lib.php` — CORS, **проверка Firebase ID-токена (RS256 по сертификатам Google)** +
    сверка email с `allowed_emails`; авторизация в Alfa (email+api_key, токен кэшируется ~50 мин);
    вызовы `/v2api/{branch}/{entity}/index`.
  - `config.example.php` — шаблон (hostname `proznanie4eee.s20.online`, email, api_key, branch,
    allowed_emails, allowed_origins). **Реальный `config.php` — только на сервере, в git и в
    автозаливку НЕ попадает** (`.gitignore` + `exclude **/config.php` в deploy.yml).
  - `.htaccess` — прячет config.php от веба + пробрасывает Authorization в PHP.
  - `README.md` — инструкция по установке.
- **Клиент (index.html):** в модель ребёнка добавлены `phone`/`alfaId` (`_blankKid`). Кнопка
  **«📞 Контакты из Альфы»** на вкладке «Расписание» (видна только админу). Тянет учеников
  через прокси (шлёт Firebase idToken в `Authorization: Bearer`), сопоставляет по ФИО
  (`_normName` = Фамилия Имя), показывает: проставим телефон / неоднозначные (выбор) /
  не найдено. По кнопке проставляет `phone`+`alfaId` всем совпавшим и пушит в Firebase.
  Адрес прокси — константа `ALFA_PROXY` (абсолютный на app.proznanie.club, чтобы работал и
  с github.io через CORS). Функции: `openAlfaSync/alfaFetch/alfaBuildMatch/alfaRenderMatch/alfaApply`.

**❗ЧТО ДОЛЖНА СДЕЛАТЬ ЖАННА, чтобы заработало (без этого прокси вернёт «config.php не найден»):**
1. На HostFly (файловый менеджер) зайти в `finmodel/api/alfa/` (появится после автозаливки).
2. Скопировать `config.example.php` → `config.php`.
3. Вписать `email` и `api_key` пользователя AlfaCRM (Alfa → Настройки → Права доступа → API-ключ).
   ⚠️ Ключ вводит САМА, в чат/код не присылать.
4. Войти в модель, открыть `https://app.proznanie.club/finmodel/api/alfa/?action=branches` —
   вернётся список филиалов с id → вписать нужный `branch` в `config.php`.
5. Нажать «📞 Контакты из Альфы» → «Загрузить из Альфы».
